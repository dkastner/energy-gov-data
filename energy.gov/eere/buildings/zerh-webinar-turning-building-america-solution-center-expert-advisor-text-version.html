And so we pull out the information from all that vast content in the Solution Center that you would need most readily to apply proven innovations. And then we attach a simple interface, so you can pull it out very quickly and much more easily than in the old past system. And, as Building America continues to generate more and more content, what will happen is you'll have more and more resources available to you. And the most important thing is, as we generate new content, the taxonomy of the engine behind the Solution Center allows us, when we introduce newcontent, to pull out old, out-of-date content and to in addition complement what we have that's there already that's still relevant. So the key here is that when you use the information in the Solution Center, you know you're getting the most up-to-date content and it's not content you have to think, "is it no longer valid or not?"

Sam Rashkin:
Well, the first thing I'm going to say on that question is, it has been a big point of interest for us, at least early when the Building America Solution Center was released. One of the largest user groups that we tracked for the Solution Center were code officials and related code stakeholders. So clearly PNNL has done a good job reaching out to that community of users, the code arena, to be aware of this information source. But the Solution Center itself is not a vehicle for influencing code. It's something that we want to make sure that stakeholders know is available to them. Because often what you have to do when engaging a code official is to document precedence or to show that there's a credible source for the practice or technology you are seeking to use that may be a question for the code official. So rather than just being, OK, here's my take on this, you're able to pull a DOE-referenced document and solution, to bring that to the official. With that said, what we do have at the Building America program is something called the Code and Standards Integration Initiative, which is CSI, oddly enough. And the purpose of that initiative is to identify areas where building science best practices might need additional integration and support with code officials, so they know how to address it, how to recognize it or prove it. And examples would be, for instance, what do you do with the marriage joint in multifamily housing where the air-sealing best practice might be at conflict with some of the fire restrictions, or fire code restrictions? And so, working with that would be an issue. We're trying to work with code officials on accepting let's say plastic as ductwork in attics, and again, address fire requirements in that situation. So there's a whole array of issues and expert meetings that have been held to identify opportunities to be more effective.
